# 03 — YAML-инфоконвейер

**Тема каталога:** #53 (приоритет А-14)
**Уровень готовности:** ★★★★

## Что это

Методологический подход: **информационные пайплайны описываются в YAML**, исполняются Python/Airflow/Prefect. Это **способ движения информации между уровнями пирамиды знаний**.

См. техническую сторону в `05_software_automation/06_yaml_infoconveyor`.

## Зачем это методология

YAML-пайплайн — это не только техника, но и **способ мышления о знаниях**:

1. **Источники** — где берутся данные
2. **Извлечение** — что важно
3. **Нормализация** — общий формат
4. **Анализ** — что значит
5. **Хранение** — куда сохранить
6. **Публикация** — куда выложить

## Связь с пирамидой знаний

```
Источник → Практический уровень (сырые данные)
   ↓ pipeline (извлечение)
Оперативный уровень (структурированные)
   ↓ pipeline (анализ, тренды)
Тактический уровень (выводы)
   ↓ pipeline (синтез, цели)
Стратегический уровень (принципы)
```

Каждый «↓» — это YAML-пайплайн.

## Шаблоны пайплайнов

### Пайплайн 1: Daily Knowledge Digest
Ежедневная подборка новостей по теме → email

### Пайплайн 2: Document Indexing
PDF → OCR → NER → ChromaDB → доступно в Bonsai

### Пайплайн 3: Conversation Archive
Разговор с ChatGPT → парсинг → темы → структуры

### Пайплайн 4: Trend Analysis
Множество источников → классификация → тренды → отчёт
