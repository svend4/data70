# Соглашения об именовании

Чтобы новая структура оставалась навигируемой, придерживаемся набора правил.

## Папки кластеров

- Префикс из двух цифр + подчёркивание + краткое имя `snake_case`
- `00_index`, `01_social_law`, `02_ai_agents` … `10_health_accessibility`, `99_archive_links`
- Цифра 99 зарезервирована для служебного

## Подпапки тем

- Префикс из двух цифр (порядок внутри кластера) + подчёркивание + краткое имя
- Пример: `01_persoenliches_budget`, `09_dat_rag`
- Цифра отражает либо приоритет, либо логический порядок чтения

## Файлы внутри подпапки

| Имя | Назначение | Обязательность |
|-----|------------|----------------|
| `README.md` | Что это, зачем, статус | Обязателен |
| `concept.md` | Концепция, теория, мета | По необходимости |
| `mvp.md` | План минимального продукта | Только для проектов с MVP |
| `roadmap.md` | Дорожная карта | Для крупных направлений |
| `funding.md` | Гранты, инвесторы, бюджет | Для коммерческих проектов |
| `risks.md` | Юридические, технические, рыночные риски | Для коммерческих проектов |
| `sources.md` | Ссылки на исходные разговоры и анализы | По необходимости |
| `examples/` | Конкретные примеры (письма, шаблоны, код) | По необходимости |

## Карточные файлы кластера

Каждый кластер `XX_*/` содержит:
- `README.md` — общее описание
- `map.md` — карта подпапок и связей
- `next_steps.md` — что развивать дальше

## Языки

- Документация — русский (основной язык автора)
- Технические термины — английский (RAG, MoE, LLM, SaaS, MVP)
- Юридические термины — немецкий с расшифровкой (Persönliches Budget, Widerspruch, SGB IX)

## Метки в README.md подпапок

Шаблон шапки:

```markdown
# <название>

**Тема каталога:** #N (если применимо)
**Кластер A-J:** X
**Беседа-источник:** #номер из chat_export
**Уровень готовности:** ★ … ★★★★★
**Приоритет:** А (можно сейчас) / Б (1-2 года) / В (3+ лет)
**Связи:** #X, #Y, #Z (внутри 88 тем)
```

## Запрет

- Не создавать файлы с именем `index.md` (использовать `README.md`)
- Не использовать пробелы и кириллицу в именах папок и файлов
- Не делать вложенность глубже 4 уровней (`monorepo/XX/YY/ZZ/file.md`)
