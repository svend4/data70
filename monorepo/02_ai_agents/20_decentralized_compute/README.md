## Что это

**Децентрализованные вычисления** — сети, в которых клиенты «арендуют» CPU/GPU у владельцев. В отличие от Exo (LAN), эти сети работают **через интернет**, с криптовалютной оплатой.

**Майнинг сценариев** — авторская идея: «майнить» не криптовалюту, а реальные HTTP-запросы и webhook-сценарии.

## Эталонные сети

| Сеть | Тип задач | Оплата |
|------|-----------|--------|
| **Akash** | Контейнерные нагрузки | AKT |
| **Golem** | Batch-вычисления | GLM |
| **iExec** | Off-chain вычисления | RLC |
| **Render** | 3D-рендеринг + AI | RNDR |
| **Bittensor** | Обучение моделей | TAO |
| **Fluence** | Облачные функции | FLT |
| **NEAR (BOS)** | Web3 фронтенд | NEAR |
| **Internet Computer (ICP)** | ИИ в блокчейне | ICP |

## Майнинг сценариев — авторская идея

Анализ показал: эти сети принимают batch-задачи, а не event-driven сценарии. Поэтому нужна **прослойка-оркестратор** (n8n, Node-RED на VPS), которая:
1. Принимает webhook
2. Решает: локально или передать в Akash/Golem
3. Возвращает результат пользователю

Это **гибридная модель**: дешёвая платформа + децентрализованные вычисления.

## Применение в проектах автора

- **Цифровой Караван** — стейкинг токенов CARAVAN привязан к реальным сценариям
- **n8n-маркетплейс** — сценарии можно запускать децентрализованно
- **AI-крипто-экосистема** (`06_business_funding/09_token_monetization`) — связка

## Открытые проблемы

1. **Latency** — децентрализованные сети медленнее, чем AWS Lambda
2. **Композиция** — отсутствует сложная оркестрация
3. **Trust** — нужны доказательства корректного исполнения (zk-proofs)
4. **UX** — для среднего пользователя слишком сложно
