# 04 — Model Context Protocol (MCP)

**Кластер A-J:** B
**Уровень готовности:** ★★★★

## Что это

**Model Context Protocol (MCP)** — открытый протокол связи LLM-приложений с инструментами и источниками данных. Разработан Anthropic в 2024. Стандартизирует то, что раньше делалось ad hoc через function calling.

Аналогия: MCP для агентов — это как HTTP для веба или USB для периферии.

## Архитектура

```
MCP Host (Claude Desktop / Claude Code / Cursor / IDE)
    ↓
MCP Client (внутри хоста)
    ↓ JSON-RPC over stdio / SSE / WebSocket
MCP Server (отдельный процесс)
    ├── Tools — функции, которые модель может вызвать
    ├── Resources — данные, которые модель может прочитать
    └── Prompts — шаблоны, которые модель может использовать
```

## Что можно сделать MCP-сервером

| Сервер | Что предоставляет |
|--------|-------------------|
| filesystem | чтение/запись файлов |
| github | issues, PR, файлы репозитория |
| postgres | SQL-запросы |
| brave-search | веб-поиск |
| memory | долговременная память |
| **СОБСТВЕННЫЕ:** | |
| `sgb-rag` | поиск по корпусу SGB |
| `widerspruch-gen` | генератор Widerspruch |
| `pflege-calc` | калькулятор Pflegeassistenz |
| `master-dossier` | работа с личным делом |

## Применение в проектах автора

1. **SozialPlaner SaaS** — каждый модуль = MCP-сервер, легко комбинируется
2. **НейроОС** — все микроагенты говорят через MCP
3. **CareMate Hub** — медицинские данные через MCP-сервер с шифрованием
4. **Telegram-бот** — переиспользует те же MCP-серверы что и SaaS

## Преимущества для автора

- **Переиспользование** — один MCP-сервер работает в Claude Desktop, своём боте, в SaaS
- **Open standard** — нет vendor lock-in
- **Композиция** — комбинировать собственные и сторонние серверы
- **Тестируемость** — каждый сервер тестируется изолированно
