# 01 — Internet Function OS / «НейроОС»

**Источник:** разговор #5 — 1766 сообщений, 4.1 МБ (самый проработанный)
**Категория:** суперпроект
**Уровень готовности:** ★★★★ (полное ТЗ + MVP Backlog)

## Что это

**Internet Function OS (IFOS)** — концепция новой архитектуры программного обеспечения, в которой каждое приложение — это **атомарная функция** со входами, выходами, зависимостями и описанием рисков. Эти функции собираются в **Blueprint** — автоматизации для конкретных задач.

Цель: решить проблему «50 000+ WordPress-плагинов и 400+ Make.com-коннекторов, но нет единого реестра».

## Архитектура (по слоям)

```
┌─────────────────────────────────────────┐
│   AI-LAYER (агенты, помощники)          │
├─────────────────────────────────────────┤
│   TRUST LAYER (риски, право, безопасн.) │
├─────────────────────────────────────────┤
│   BLUEPRINTS (готовые связки)            │
├─────────────────────────────────────────┤
│   COMPOSER (интерфейс сборки)            │
├─────────────────────────────────────────┤
│   VOCABULARY / SCHEMA (онтология)        │
├─────────────────────────────────────────┤
│   FUNCTION REGISTRY (атомарные блоки)    │
└─────────────────────────────────────────┘
```

## Аналогия «паспорт блока»

Каждое приложение/плагин/функция получает **«паспорт»** — карточку со структурированными полями:
- Имя
- Назначение
- Входы (типы данных, источники)
- Выходы
- Зависимости
- Стоимость (в €, минутах, ресурсах)
- Риски (правовые, безопасности, бизнес-логики)
- Совместимость с другими блоками
- Лицензия

## Ключевые компоненты (полная реализация)

### 1. Function Registry (`14_function_registry`)
- CSV с 60 готовыми записями для WordPress / Make / n8n
- Каждая запись = «паспорт» функции
- API для добавления/поиска функций

### 2. Vocabulary / Schema
- Стандартные типы данных (Lead, Order, Document, User…)
- Маппинг между блоками с автоматической трансформацией

### 3. Composer
- Drag-and-drop интерфейс сборки автоматизаций
- Подсказывает совместимые блоки
- Предупреждает о рисках

### 4. Blueprints (`15_blueprints_oneclick`)
- 10 готовых «one-click» автоматизаций:
  - Лиды → CRM → email
  - Заказы → склад → доставка
  - Бэкапы файлов → S3 → отчёт
  - API-интеграция типичных сервисов

### 5. Trust Layer
- Юридические проверки (GDPR, AGG)
- Отслеживание рисков утечек
- Compliance-отчёты

### 6. AI Layer
- Помощник в Composer (рекомендации блоков)
- Автогенерация Blueprint по описанию
- Обнаружение аномалий в работе

## MVP Backlog (`16_mvp_backlog`)

Эпики:
1. Function Registry — БД и API (1 мес.)
2. Vocabulary / Schema — онтология (1 мес.)
3. Composer UI — drag-and-drop (2-3 мес.)
4. 10 Blueprint демо (1 мес.)
5. Trust Layer — базовые проверки (1 мес.)
6. AI-помощник — выбор блоков (1 мес.)

Всего: **6-9 месяцев на MVP, бюджет €100-150K**.
