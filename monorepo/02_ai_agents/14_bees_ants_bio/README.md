# 14 — Биологическая модель ИИ: «Пчёлы и Муравьи»

**Кластер A-J:** A
**Категория:** изобретение / концепция
**Уровень готовности:** ★★★★

## Что это

**Авторская концепция** двухуровневой мультиагентной архитектуры:

- **Пчелиный слой** — тысячи микроагентов-исполнителей. Каждая «пчела» делает узкую задачу: сбор нектара (данных), пыльца (метаданных), строительство сот (структуры). Простые, быстрые, заменимые.
- **Муравьиный слой** — надзорные агенты-контролёры. Распределяют задания, проверяют качество, управляют конфликтами, оптимизируют ресурсы.

## Биологическое обоснование

| Свойство | Пчёлы | Муравьи |
|----------|-------|---------|
| Размер | Тысячи | Сотни |
| Задача | Сбор / производство | Координация / разведка |
| Жизненный цикл | Короткий | Длинный |
| Стоимость | Низкая | Высокая |
| Заменимость | Высокая | Низкая |

Это формализация естественной эволюционной стратегии: «много дешёвых исполнителей + меньше дорогих координаторов».

## Архитектурные таблицы

### Пчёлы — типы и роли
| Роль | Функция | Пример в SozialPlaner |
|------|---------|----------------------|
| Сборщица | Парсинг входящего | Парсер Bescheid |
| Строитель | Структурирование | Заполнение Master Dossier |
| Защитница | Валидация | Проверка корректности Widerspruch |
| Кормилица | Распределение | Доставка результата пользователю |

### Муравьи — типы и роли
| Роль | Функция | Пример |
|------|---------|--------|
| Разведчик | Обнаружение новых задач | Мониторинг Bescheid |
| Координатор | Распределение работы | Назначение пчёл на задачи |
| Контролёр | QA пчёл | Проверка качества |
| Архитектор | Оптимизация | Настройка пайплайнов |

## Где это применяется

- **НейроОС** — пчелиный слой = микроагенты, муравьиный = планировщики
- **SozialPlaner** — пчёлы = парсеры/генераторы, муравьи = валидаторы
- **CareMate** — пчёлы = модули мониторинга, муравьи = диспетчеры
- **Newsroom** — пчёлы = pipeline-этапы, муравьи = редактор+фактчекер

## Связь с MoE

MoE (`06_sparse_moe`) — пчёлы внутри одной модели. Эта концепция — пчёлы как отдельные агенты. Архитектурно похоже, но реализации разные.
