# 11 — Принцип числа конечностей робота

**Тема каталога:** #59
**Уровень готовности:** ★★★★

## Что это

Авторская **методология классификации роботов по числу конечностей** и связанной функциональной сложности.

Идея: количество конечностей = верхний предел сложности задач. Чем больше степеней свободы, тем больше задач, но и тем дороже + сложнее в управлении.

## Классификация

| Конечностей | Категория | Пример |
|-------------|-----------|--------|
| **0** | Без конечностей | Шар (`05_where_bot_spiral` — спираль) |
| **1** | Хобот / змея | Soft-роботы, манипуляторы |
| **2** | Руки или ноги | Двуногий, манипулятор-стенд |
| **3** | Триподы | Стационарные манипуляторы |
| **4** | Базовый | Собака (Spot), стол с 4 ногами |
| **5** | «Морская звезда» | Pentagrammacopter |
| **6** | Насекомое | Hexapods (Spider) |
| **8** | Паук | Octopus / 8-ноги |
| **много** | Многоножка / спираль | Soft snake-роботы |

## Связь с задачами

| Задача | Минимум конечностей | Идеал |
|--------|---------------------|-------|
| Движение по плоскости | 0-2 (колёса) | 4 |
| Лестница | 2 (ноги) | 4-6 |
| Перенос предметов | 1 (рука) | 2 |
| Манипуляция тонким | 2 (двусторонняя) | — |
| Жидкости | 2 (рука + ёмкость) | — |
| Шитьё / готовка | 2 (бимануальная) | — |
| Игра в шахматы / реверси | 1 (рука) | — |

## Применение в проекте «Рой 15 зверей»

В `01_15_robot_zoo` каждый робот имеет **минимально нужное число конечностей** для своей задачи:

- **Снегирь** (доставка) — 0 ног + 4 пропеллера + 1 гриппер
- **Бобр** (уборка) — 0 ног + щётки
- **Барсук** (сортировка) — 4 ноги + 1 рука
- **Стриж** (курьер) — 0 ног + 4 пропеллера
- **Енот** (манипуляция) — 4 ноги + 2 руки

Принцип: **никогда не больше конечностей, чем требует задача**. Это даёт:
- Низкую цену
- Высокую надёжность
- Простоту программирования
- Меньше энергии
