# 03 — Архитектура по У-Синь (5 стихий)

**Уровень готовности:** ★★★

## Что это

**У-Синь** (五行) — китайская философская система пяти стихий: Вода, Дерево, Огонь, Земля, Металл. В CareMate Hub автор использует её как **архитектурный паттерн** проектирования.

## Маппинг стихий → модули CareMate

| Стихия | Природа | Архитектурная роль |
|--------|---------|--------------------|
| **Вода** | Поток, ввод | Сенсоры, потоки данных извне |
| **Дерево** | Рост, анализ | ML/NLP анализ входящего |
| **Огонь** | Действие, реакция | Немедленные действия (SOS, вызовы) |
| **Земля** | Накопление | Долговременная память, история |
| **Металл** | Структура, граница | Правила, валидация, безопасность |

## Циклы порождения и преодоления

В У-Синь стихии связаны:

**Цикл порождения** (что что создаёт):
- Вода → Дерево (поток питает анализ)
- Дерево → Огонь (анализ приводит к действию)
- Огонь → Земля (действие сохраняется в памяти)
- Земля → Металл (память формирует правила)
- Металл → Вода (правила фильтруют поток)

**Цикл преодоления** (что что ограничивает):
- Вода → Огонь (поток данных ограничивает поспешные действия)
- Огонь → Металл (срочное действие может нарушить правила)
- Металл → Дерево (правила ограничивают анализ)
- Дерево → Земля (анализ переписывает память)
- Земля → Вода (память фильтрует поток)

## Применение

В CareMate Hub эти циклы реализованы как:

1. **Порождающий цикл** — нормальный поток обработки события
2. **Преодолевающий цикл** — система проверки и ограничений (например, правило «огонь не действует без воды» = «нет действия без подтверждённых данных»)

## Зачем это нужно

1. **Дисциплина проектирования** — каждый модуль имеет чёткую роль
2. **Тестируемость** — циклы дают набор сценариев для unit-тестов
3. **Маркетинг** — необычная концепция с восточной философией
4. **Связь с темой #67** (Звезда-Агент в Соте) — ещё одна геометрическая система
