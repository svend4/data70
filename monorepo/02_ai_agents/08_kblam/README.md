# 08 — KBLaM (Knowledge Base augmented Language Model)

**Темы каталога:** #5, #31
**Кластер A-J:** A
**Уровень готовности:** ★★★

## Что это

**KBLaM** (Microsoft Research, 2024) — архитектура, при которой база знаний **встроена внутрь модели**, а не подаётся через RAG. Это не контекстное окно с фактами, а отдельный слой памяти, к которому модель «обращается» как к части своего весового пространства.

Отличается от RAG тем, что не требует отдельного поиска — знания доступны мгновенно как часть инференса.

## Почему это интересно

1. **Скорость** — нет отдельного шага поиска
2. **Целостность** — модель не теряет факты между запросами
3. **Обновляемость** — БЗ обновляется без переобучения
4. **Приватность** — БЗ может быть локальной

## Применение

- **SozialPlaner** — корпус SGB как KBLaM-слой → модель «знает» все параграфы
- **CareMate** — медицинская БЗ как часть модели
- **НейроПортал** — нишевые БЗ для каждого домена

## Сравнение с RAG (тема #11)

| Признак | RAG | KBLaM |
|---------|-----|-------|
| Поиск | Внешний (vector DB) | Встроенный |
| Скорость | Latency на поиск | Без задержки |
| Контекстное окно | Тратится | Свободно |
| Обновление | Лёгкое | Сложнее |
| Зрелость | Производство | Ресёрч |
