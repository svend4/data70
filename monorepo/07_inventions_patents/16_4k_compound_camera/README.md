# 16 — Составная 4K-камера (28 матриц 600×600) — патентная карточка

**Источник:** разговор #795 — 522 сообщения, 1.9 МБ
**Категория:** A (Оптика)
**Новизна:** ★★★★
**Применимость:** ★★★
**Статус:** Расчёт готов

## Суть

**4K-камера** из массива **28 малых матриц 600×600 пикселей**. Программная сшивка в 4K-изображение. Каждая матрица — отдельный сегмент с собственной экспозицией, фокусом, временем затвора.

## Что отличает

| Аналог | Подход | Чем отличается |
|--------|--------|----------------|
| **Стандартная 4K-матрица** | Одна большая | Сложно ремонтировать, дорого |
| **Multi-camera arrays** (телефоны) | 2-3 матрицы | 28 для полного 4K |
| **Light field cameras** (Lytro) | Микролинзы | Иной принцип |
| **Авторская концепция** | 28 матриц 600×600 + сшивка | Massive parallel + ремонтопригодность |

## Что заявляется

1. **Геометрия массива** 7×4 с перекрытием 5-10%
2. **Метод сшивки** — OpenCV + ML-коррекция швов
3. **Метод калибровки** — заводская + динамическая в полёте
4. **Управление каждым сегментом** — индивидуальная экспозиция

## Преимущества

1. **Ремонтопригодность** — заменить один сегмент дешевле
2. **Резервирование** — отказ одного, остальные продолжают
3. **Динамический диапазон** — каждый сегмент со своей экспозицией → HDR
4. **Цена** — 28 малых матриц дешевле одной большой 4K
5. **Гибкость** — изогнутая поверхность массива (180° съёмка)

## Применение

- **Дроны SkyMediaHub** — широкоугольная съёмка
- **Системы наблюдения** — массивы на куполах
- **Автомобильное видение** — массивы вокруг машины
- **VR-рекордеры** — 360° съёмка

## Стратегия

1. **POC на 4 матрицах OV5647** (Raspberry Pi camera)
2. **Сшивка софт** на основе OpenCV
3. **Подача DPMA** при подтверждении
4. **Партнёрство Fraunhofer IPMS** — оптика чипов
