# 09 — KBLaM как методология

**Темы каталога:** #5, #31
**Уровень готовности:** ★★★

## Что это

**KBLaM** (Knowledge Base augmented Language Model, Microsoft Research, 2024) — не только техника, но и **методологический подход**: знание встраивается **внутрь** модели как часть её весов, а не подаётся через RAG.

## Методологическое отличие от RAG

| Подход | Что |
|--------|-----|
| **RAG** | Знание во внешнем индексе, ищется при каждом запросе |
| **KBLaM** | Знание в модели, доступно мгновенно |
| **Гибрид** | Частые знания в модели, редкие — в RAG |

## Зачем как методология

Концептуально, KBLaM — это **«встроенная мудрость»** vs **«доступ к библиотеке»**:

- RAG — модель ходит в библиотеку
- KBLaM — модель «знает» как эксперт

Это меняет архитектуру решения:
- Нет latency на поиск
- Нет потерь контекста
- Нет risk на качество поиска
- Сложнее обновлять

## Применение в проектах автора

| Проект | Что встраивать в KBLaM |
|--------|------------------------|
| SozialPlaner | Все параграфы SGB IX/XII (фиксированные) |
| CareMate | Медицинские протоколы (стабильные) |
| НейроПортал | Базовая ниша (юридическая, медицинская) |

При этом **новости и обновления остаются в RAG**.
