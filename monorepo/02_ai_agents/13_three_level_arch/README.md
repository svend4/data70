# 13 — Трёхуровневая архитектура нейросетей

**Темы каталога:** #32, #56
**Кластер A-J:** A/B/H
**Уровень готовности:** ★★★★

## Что это

**Авторская мета-архитектура** — деление любой ИИ-системы на три уровня:

1. **Оперативный** — рутинные операции (быстро, дёшево, локально)
2. **Тактический** — координация, планирование (среднее)
3. **Стратегический** — целеполагание, ценности (дорого, редко, в облаке)

Это применимо к:
- Архитектуре нейросети (слои)
- Архитектуре системы (микросервисы)
- Архитектуре агентов (исполнитель/координатор/планировщик)
- Архитектуре управления знаниями (см. `08_knowledge_methodology/01_pyramid_4_levels`)

## Конкретное применение в НейроОС

```
СТРАТЕГИЧЕСКИЙ слой   ← Claude / GPT-5 (редкие, дорогие запросы)
     ↓
ТАКТИЧЕСКИЙ слой      ← Llama 3.1 70B (планирование задач)
     ↓
ОПЕРАТИВНЫЙ слой      ← Llama 3.1 8B / Phi-3.5 (быстрые рутинные шаги)
```

## Конкретное применение в SozialPlaner

```
СТРАТЕГИЧЕСКИЙ — Claude API (составление иска в Sozialgericht)
     ↓
ТАКТИЧЕСКИЙ — Mixtral 8x7B (генерация Widerspruch)
     ↓
ОПЕРАТИВНЫЙ — Llama 8B Q4 (поиск в SGB, простые ответы)
```

## Принцип роутинга

Для каждого запроса определяется уровень сложности:
- **Дёшевый** (поиск факта) → оперативный
- **Средний** (генерация документа) → тактический
- **Дорогой** (юридическая стратегия) → стратегический

Минимизация затрат + максимизация качества.
