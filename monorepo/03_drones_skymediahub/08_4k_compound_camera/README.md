# 08 — Составная 4K-камера для дронов (28 матриц 600×600)

**Источник:** разговор #795 — 522 сообщения, 1.9 МБ (один из самых проработанных)
**Уровень готовности:** ★★★★ (детальный расчёт)

## Что это

**4K-камера**, собранная не из одной большой матрицы, а из **массива 28 малых матриц 600×600** пикселей.

Принцип: вместо дорогой и хрупкой большой матрицы — **сетка дешёвых маленьких** с программной сшивкой и взаимной коррекцией.

## Зачем это нужно

1. **Ремонтопригодность** — сломавшийся сегмент заменяется отдельно
2. **Стоимость** — мелкие матрицы 600×600 в сотни раз дешевле большой 4K
3. **Гибкость** — изогнутая поверхность массива, угол обзора 180-270°
4. **Резервирование** — при отказе сегмента остальные продолжают работать
5. **Динамический диапазон** — каждый сегмент со своей экспозицией

## Расчёт массива

```
4K (3840 × 2160) ÷ 600 ≈ 7 × 4 (горизонталь × вертикаль)
                           = 28 матриц
```

Сетка 7×4. Перекрытие на стыках 5-10% для сшивки.

## Технические вопросы

| Вопрос | Решение |
|--------|---------|
| Сшивка | OpenCV stitching + ML-коррекция швов |
| Калибровка | Заводская + продолжающаяся «in-flight» |
| Параллакс | Для близких объектов нужен depth-map |
| Синхронизация затворов | Жёсткая через FPGA |
| Пропускная способность | 28 × MIPI CSI-2 на FPGA-агрегатор |

## Применения

1. **Дроны** — широкоугольная съёмка с разной экспозицией
2. **Системы наблюдения** — массивы на куполах
3. **Автомобильное видение** — массивы вокруг машины
4. **VR-рекордеры** — 360° съёмка
