# 01 — AI-Newsroom: автоматизация журналистики

**Тема каталога:** #80
**Источник:** разговор #604 + ~10 смежных
**Уровень готовности:** ★★★★

## Что это

**Сквозная AI-цепочка** для редакций: от мониторинга источников до публикации готовых статей.

В архиве автора уже есть:
- Python-код с Haystack + pgvector
- Таблица AI-инструментов для каждого этапа
- Концепция «Археологии идей» — 16-секционный документ

## 8 этапов цепочки

См. подробную карту в `09_media_newsroom/map.md`:

1. Мониторинг (RSS, соцсети, госреестры)
2. Извлечение (OCR, ASR, скрейпинг)
3. NER + нормализация
4. Векторное хранение
5. RAG — поиск
6. Генерация черновика
7. Фактчекинг
8. Публикация

## Технологический стек

| Слой | Технологии |
|------|-----------|
| Мониторинг | RSS-Parser, Tweepy, Mastodon-API |
| OCR | Tesseract / EasyOCR |
| ASR | Whisper |
| NER | spaCy (de_core_news_lg) |
| Embeddings | mxbai-embed-large |
| Vector DB | pgvector / Qdrant |
| RAG | Haystack 2.0 / LlamaIndex |
| Генерация | Claude API + локальный Llama |
| Шаблоны | Jinja2 |
| Фактчекинг | Custom + LLM-self-check |
| CMS | WordPress / Strapi / собственный |
