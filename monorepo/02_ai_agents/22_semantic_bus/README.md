# 22 — Семантический автобус для смартфонов / AI Runtime

**Тема каталога:** #85
**Кластер A-J:** B
**Уровень готовности:** ★★★

## Что это

**AI Runtime / Semantic Bus** — единая системная служба в мобильной ОС, которая предоставляет нейросетевые модели всем приложениям через общее API.

Эталоны:
- **Android AICore** (Google) — Gemini Nano для всех приложений
- **Apple Intelligence** — Apple Foundation Models
- **Авторская концепция** — расширение: векторное хранилище + граф API + микроагенты

## Что добавляет автор к существующим AI Runtime

1. **Векторное хранилище** — общая ChromaDB на устройстве для всех приложений
2. **Граф API** — приложения публикуют семантические интерфейсы, не просто данные
3. **Микроагенты** — приложения могут запускать друг друга через шину
4. **Кеш моделей** — единый, чтобы не дублировать веса

## Концепция «семантический автобус»

```
  ┌─────────────┐
  │  App A      │ публикует API "search_documents" + векторы
  └──────┬──────┘
         ↓
  ┌─────────────────────────────────────┐
  │  Semantic Bus (системная служба)     │
  │  ├── Embedding cache                 │
  │  ├── Vector DB (общий ChromaDB)     │
  │  ├── Model cache (Llama, Phi)        │
  │  └── API graph (кто что предоставляет)│
  └─────┬───────────────────────────────┘
         ↓
  ┌─────────────┐
  │  App B      │ потребляет API "search_documents" семантически
  └─────────────┘
```

## Зачем это нужно

1. **Память** — не дублировать модель в каждом приложении (для Llama 3 8B Q4 это 5 ГБ)
2. **Скорость** — модель прогрета, доступна мгновенно
3. **Композиция** — приложения собираются как блоки
4. **Приватность** — данные не покидают устройство

## Применение

- **НейроОС** — Semantic Bus как ядро ОС
- **Forth-Рой** — мини-Bus в роботе для обмена «смыслами» с другими роботами
- **CareMate** — общий слой для медицинских приложений
- **Внутри SozialPlaner** — модули общаются через Bus
