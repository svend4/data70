# 11 — Фактчекинг

**Уровень готовности:** ★★★

## Что это

**Этап проверки** сгенерированных статей перед публикацией. Цель — выявить:
- Цитаты, которых не было
- Цифры, которые «выдуманы»
- Даты, которые не подтверждаются
- Связи событий, которых нет

## Уровни фактчекинга

### Уровень 1: Self-check LLM
LLM отвечает на свой собственный текст:
- «Какие цитаты есть в этом тексте?»
- «Подтверждается ли цитата 1 источниками?»

### Уровень 2: RAG-проверка
- Каждое утверждение проверяется по базе
- Если confidence < threshold → пометить

### Уровень 3: Human-in-the-loop
- Журналист видит сомнительные места
- Ручное подтверждение

## Технологии

- **Logic-LLM** — символическая проверка
- **Searcher-RAG** — повторный поиск
- **Annotator UI** — для ручной проверки

## Применение

- AI-newsroom — обязательный этап перед публикацией
- SozialPlaner — проверка генерируемых документов
- CareMate — проверка медицинских сводок
- НейроПортал — модерация UGC
